Frontend

Interface Utilisateur

Web + Mobile
React 18 + TypeScript
Écosystème riche, composants réutilisables, typage fort indispensable pour la logique financière complexe.
TailwindCSS
Développement UI rapide, design system cohérent, bundle optimisé. Idéal pour équipe réduite.
Recharts + AG Grid
Recharts pour les graphiques KPIs. AG Grid pour les tableaux budgétaires style Excel — expérience familière pour les DAF.
React Native (V2)
App mobile pour dashboard CEO. Partage de code avec le web frontend.
⚙️
Backend

Services Métier

API REST + Microservices
Node.js + NestJS
Architecture modulaire, injection de dépendances, TypeScript natif. Parfait pour les services CRUD (Auth, Budget, Scénarios, Notifications).
Python + FastAPI
Pour le moteur de calcul financier et l'import Excel. Pandas pour la manipulation de données, NumPy pour les calculs matriciels.
Kong API Gateway
Rate limiting, auth centralisée, logs, routing vers les microservices. Open-source et battle-tested.
GraphQL (V3)
Envisageable en V3 pour les requêtes de reporting complexes (multi-entités, consolidation).
🗄️
Base de données

Persistance & Cache

Données financières & sessions
PostgreSQL 16
ACID compliance indispensable pour les données financières. Support JSON, types Decimal précis (pas de float!), Row-Level Security pour le multi-tenant.
Redis 7
Cache des sessions JWT, job queue (BullMQ), pub/sub pour les WebSockets de progression d'import en temps réel.
Prisma ORM
Migrations typées, schéma as code, auto-génération des types TypeScript. Zéro SQL brut dangereux.
TimescaleDB (V3)
Extension PostgreSQL pour les séries temporelles — utile si on ajoute de l'IA prédictive sur les KPIs.
🔐
Sécurité

Auth & Conformité

Données financières sensibles
JWT + Refresh Token
Access token 8h, refresh token 30j. Stockage en mémoire côté client (jamais localStorage). Rotation automatique.
bcrypt (argon2)
Hachage des mots de passe. Argon2id recommandé — résistant aux attaques GPU, standard OWASP 2024.
Row-Level Security
Isolation multi-tenant au niveau PostgreSQL. Même si un bug applicatif survient, un tenant ne voit jamais les données d'un autre.
2FA TOTP (V2)
Authentification à 2 facteurs pour les comptes DAF/Admin. Google Authenticator compatible.
⚡
Async

Traitement asynchrone

Jobs longue durée & temps réel
BullMQ (Redis)
Queue de jobs pour import Excel, recalculs financiers, génération de rapports PDF. Retry automatique, monitoring intégré.
Socket.io
WebSocket pour la progression en temps réel (import, calcul de scénario). Fallback long-polling si réseau instable.
node-cron
Recalcul automatique des KPIs chaque nuit. Envoi des alertes hebdomadaires. Clôture automatique des périodes.

Infrastructure

Hébergement & CI/CD

Optimisé Afrique de l'Ouest
DigitalOcean (Lagos)
Datacenter Lagos, Nigeria — latence < 30ms pour l'Afrique de l'Ouest. Tarifs accessibles, Kubernetes managé.
Docker + Kubernetes
Conteneurisation de chaque microservice. Scaling horizontal automatique en pic de charge (clôture budgétaire fin de mois).
Cloudflare CDN
Cache des assets statiques, protection DDoS, acceleration réseau Afrique. Gratuit jusqu'à fort volume.
AWS Cape Town (V2)
Alternative si la base client s'étend à l'Afrique australe et orientale.

💡
Principe directeur : La stack recommandée privilégie la fiabilité des données financières avant la sophistication technique. PostgreSQL avec ACID compliance et Row-Level Security garantit qu'un DAF ne verra jamais les données d'une autre organisation — même en cas de bug applicatif. Python pour le moteur de calcul garantit une précision décimale parfaite, critique pour les états financiers. L'hébergement à Lagos réduit la latence pour les utilisateurs d'Afrique de l'Ouest de 200ms à moins de 30ms, ce qui change radicalement l'expérience utilisateur sur connexions 3G/4G.
